# Architecture Brief: 数学斜坡跑

**Created:** 2026-03-03

## 技术栈

- **前端**: HTML5, CSS3, JavaScript (ES6+)
- **渲染**: Canvas 2D 上下文
- **状态管理**: 全局变量和函数
- **数据持久化**: localStorage
- **构建工具**: 无（纯静态文件）

## 文件结构

```
game-factory/games/math-slope-run/
├── index.html          # 主游戏文件（包含内联 CSS 和 JavaScript）
└── README.md           # 游戏说明（可选）
```

## 核心模块

### 1. 游戏状态管理

```javascript
const gameState = {
  score: 0,           // 当前分数
  distance: 0,        // 当前距离（米）
  difficulty: 'add',  // 当前难度：add, subtract, multiply, divide
  isGameOver: false,  // 游戏是否结束
  isPaused: false,    // 游戏是否暂停
  ball: {             // 球体状态
    x: 0,
    y: 0,
    z: 0,
    vx: 0,
    vy: 0,
    vz: 0,
    radius: 10,
    color: '#3b82f6'
  },
  obstacles: [],      // 障碍物数组
  achievements: {},   // 成就进度
  highScore: 0,       // 最高分
  hasShield: false,   // 是否有护盾
  speed: 1.0          // 当前速度倍率
};
```

### 2. Canvas 渲染系统

- **斜坡渲染**: 使用 Canvas 2D 绘制 3D 斜坡效果（透视投影）
- **球体渲染**: 绘制球体，根据状态改变颜色
- **障碍物渲染**: 绘制障碍物，根据距离调整大小
- **背景渲染**: 绘制渐变背景和路径

### 3. 控制系统

- **设备倾斜**: 使用 DeviceOrientation API 检测设备倾斜
- **点击/触摸**: 支持点击或触摸屏幕左右两侧控制方向
- **平滑移动**: 使用加速度和摩擦力实现平滑移动

### 4. 数学题系统

- **题目生成**: 根据当前难度级别生成随机数学题
  - 加法：1-20 以内的加法
  - 减法：1-20 以内的减法（确保结果为正）
  - 乘法：1-10 以内的乘法
  - 除法：1-10 以内的除法（确保整除）
- **弹出界面**: 使用模态框显示题目
- **答案验证**: 比较玩家输入与正确答案
- **反馈**: 答对显示绿色，答错显示红色

### 5. 难度递进系统

- **距离跟踪**: 每帧更新距离
- **难度切换**: 每 500 米切换一次难度
  - 0-500 米：加法
  - 500-1000 米：减法
  - 1000-1500 米：乘法
  - 1500+ 米：除法
- **速度递增**: 每 100 米，速度增加 2%

### 6. 视觉反馈系统

- **球体颜色**: 根据状态改变颜色
  - 正常：蓝色
  - 护盾激活：金色
  - 加速：红色
- **路径颜色**: 根据距离渐变
- **得分特效**: 收集分数时显示浮动分数
- **成就解锁**: 解锁成就时显示提示

### 7. 成就系统

```javascript
const achievements = [
  { id: 'distance1000', name: '长途跋涉', description: '前进1000米', target: 1000, type: 'distance' },
  { id: 'math50', name: '数学大师', description: '答对50题', target: 50, type: 'math' },
  { id: 'obstacle100', name: '障碍高手', description: '避开100个障碍物', target: 100, type: 'obstacle' }
];
```

### 8. 本地存储

- **最高分**: 保存最高分到 localStorage
- **成就进度**: 保存成就解锁状态
- **游戏进度**: 可选保存当前游戏进度（用于恢复）

## 用户界面设计

### 主游戏区域

- **顶部**: 分数、距离、当前难度显示
- **中央**: Canvas 游戏区域
- **底部**: 控制按钮（开始、暂停、重新开始）

### 数学题模态框

- **题目显示**: 大字体显示数学题
- **答案输入**: 数字输入框
- **提交按钮**: 提交答案
- **反馈区域**: 显示正确/错误反馈

### 成就展示

- **成就列表**: 显示所有成就及其解锁状态
- **解锁提示**: 解锁成就时弹出提示

## 响应式设计

- **移动端优先**: 使用 vw/vh 单位和百分比布局
- **触摸事件**: 支持触摸和鼠标事件
- **屏幕适配**: 自动调整 Canvas 大小和游戏区域

## 性能优化

- **Canvas 优化**: 使用 requestAnimationFrame 进行渲染
- **对象池**: 重用障碍物对象，减少垃圾回收
- **碰撞检测**: 使用简单的距离检测，避免复杂计算

## 部署

游戏为纯静态文件，可直接部署到任何静态文件服务器（如 GitHub Pages）。
