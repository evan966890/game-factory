# Tech-Spec: 数学曲折跑

**Created:** 2026-03-03

## Overview

### Problem Statement

如何将热门浏览器游戏 Zigzag Runner 的简单上瘾玩法与数学教育结合，创造一个既能娱乐又能学习的教育游戏，同时保持移动端友好和单指操作。

### Solution

开发一个纯 HTML5 游戏，核心玩法是点击控制球在曲折路径上滚动，避开障碍，收集分数。每前进 100 米弹出一道数学题，答对获得加速和护盾，答错减速。游戏包含难度递进（加法→减法→乘法→除法）、视觉反馈、成就系统和本地存储最高分。

### Scope

**In Scope:**
- 核心滚动机制：点击控制球在曲折路径上滚动，避开障碍，收集分数
- 数学题生成：每前进 100 米弹出一道随机数学题，难度随距离递进
- 分数系统：基础分 + 答题奖励分
- 难度曲线：加法（0-500米）→ 减法（501-1000米）→ 乘法（1001-1500米）→ 除法（1501+米）
- 视觉反馈：球体颜色变化、路径颜色渐变、得分特效、答题正确/错误反馈
- 成就系统：前进 1000 米、答对 50 题等成就
- 本地存储：保存最高分和成就进度
- 移动端响应式设计，单指操作

**Out of Scope:**
- 多人游戏或在线排行榜
- 复杂 3D 图形或精灵图集
- 音效（可后续添加）
- 用户账户系统
- 复杂的物理引擎

### Technical Decisions

1. **单文件结构**：便于部署和分享，所有资源内联
2. **Canvas vs DOM**：使用 Canvas 绘制路径和球体，性能更好
3. **数学题生成**：使用 JavaScript 随机生成题目，确保答案为整数
4. **难度递进**：基于前进距离切换题目类型
5. **本地存储**：使用 localStorage 保存最高分和成就

## Implementation Plan

### Tasks

1. **基础结构搭建**
   - 创建 index.html 基础模板
   - 设置 Canvas 和 UI 元素（分数、距离、成就显示）

2. **核心滚动机制**
   - 实现球体生成和移动逻辑
   - 实现曲折路径生成
   - 实现碰撞检测和得分计算
   - 实现滚动动画和视觉反馈

3. **数学题系统**
   - 实现题目生成算法（加减乘除）
   - 实现答题界面和交互
   - 实现答对奖励和答错惩罚

4. **难度递进系统**
   - 根据前进距离切换题目类型
   - 调整路径复杂度和速度增加难度

5. **成就和存储系统**
   - 实现成就检测和解锁
   - 实现本地存储读写

6. **UI 和反馈**
   - 设计移动端友好的界面
   - 添加 CSS 动画和特效
   - 实现“再来一局”功能

7. **测试和优化**
   - 移动端测试
   - 性能优化
   - Bug 修复

- 游戏节奏由慢到快，保持心流体验
