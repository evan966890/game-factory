# Tech-Spec: 数学堆叠塔

**Created:** 2026-03-03

## Overview

### Problem Statement

如何将经典的 Stack 堆叠游戏与数学教育相结合，创造一个既有上瘾性又有教育意义的休闲游戏？目标是让玩家在享受堆叠乐趣的同时，自然地进行数学练习，提高计算能力。

### Solution

基于 Stack 的堆叠玩法，玩家通过单指点击屏幕堆叠方块，时机判断追求完美堆叠。每堆叠5层弹出一道数学题，答对获得奖励（如额外分数、方块变宽），答错受到惩罚（如方块变窄、速度加快）。难度随层数递进：加法 → 减法 → 乘法 → 除法。游戏包含视觉反馈、成就系统和本地存储最高分。

### Scope

**In Scope:**
- 核心堆叠机制：点击屏幕堆叠方块，时机判断，追求完美堆叠
- 数学题弹出：每堆叠5层弹出一道数学题，答对奖励，答错惩罚
- 难度递进：加法 → 减法 → 乘法 → 除法，随层数解锁
- 视觉反馈：方块颜色变化，堆叠特效，得分特效
- 成就系统：堆叠50层、答对30题等成就
- 本地存储：保存最高分和进度
- 移动端优先，响应式设计
- 纯 HTML/CSS/JS，无框架依赖

**Out of Scope:**
- 多人游戏或在线排行榜
- 复杂图形或3D效果
- 音效或背景音乐
- 用户账户系统
- 复杂的设置或选项菜单

## Context for Development

### Codebase Patterns

游戏工厂采用纯 HTML/CSS/JS 技术栈，无框架依赖。所有游戏都是独立的 HTML 文件，包含内联 CSS 和 JavaScript。游戏通常包含以下结构：
- 一个主容器 div
- Canvas 或 DOM 元素用于游戏渲染
- 游戏状态管理（分数、层数、游戏状态）
- 事件监听（点击、触摸）
- 本地存储用于保存最高分

### Technical Decisions

1. **渲染方式**：使用 DOM 元素（div）而不是 Canvas，因为堆叠游戏更适合 DOM 操作，且易于实现响应式设计。
2. **数学题生成**：使用 JavaScript 随机生成数学题，根据当前难度级别调整运算符和数字范围。
3. **状态管理**：使用全局变量存储游戏状态（分数、层数、当前难度、游戏是否结束）。
4. **本地存储**：使用 localStorage 保存最高分和成就进度。
5. **响应式设计**：使用 CSS 百分比和 vw/vh 单位，确保在不同屏幕尺寸上正常显示。
