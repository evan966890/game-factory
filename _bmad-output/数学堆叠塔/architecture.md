# Architecture Brief: 数学堆叠塔

**Created:** 2026-03-03

## 技术栈

- **前端**: HTML5, CSS3, JavaScript (ES6+)
- **渲染**: DOM 元素（div）用于方块和界面
- **状态管理**: 全局变量和函数
- **数据持久化**: localStorage
- **构建工具**: 无（纯静态文件）

## 文件结构

```
game-factory/games/math-stacking-tower/
├── index.html          # 主游戏文件（包含内联 CSS 和 JavaScript）
└── README.md           # 游戏说明（可选）
```

## 核心模块

### 1. 游戏状态管理

```javascript
const gameState = {
  score: 0,           // 当前分数
  level: 1,           // 当前层数
  difficulty: 'add',  // 当前难度：add, subtract, multiply, divide
  isGameOver: false,  // 游戏是否结束
  currentBlock: null, // 当前方块
  stack: [],          // 已堆叠的方块数组
  achievements: {},   // 成就进度
  highScore: 0        // 最高分
};
```

### 2. 方块系统

- **方块生成**: 根据当前层数和难度生成方块
- **方块移动**: 使用 CSS transform 和 transition 实现平滑移动
- **堆叠检测**: 比较当前方块与下方方块的位置，计算重叠区域
- **完美堆叠**: 重叠区域超过 90% 视为完美堆叠，获得奖励

### 3. 数学题系统

- **题目生成**: 根据当前难度级别生成随机数学题
  - 加法：1-20 以内的加法
  - 减法：1-20 以内的减法（确保结果为正）
  - 乘法：1-10 以内的乘法
  - 除法：1-10 以内的除法（确保整除）
- **弹出界面**: 使用模态框显示题目
- **答案验证**: 比较玩家输入与正确答案
- **反馈**: 答对显示绿色，答错显示红色

### 4. 难度递进系统

- **层数跟踪**: 每堆叠一层，层数加一
- **难度切换**: 每 10 层切换一次难度
  - 1-10 层：加法
  - 11-20 层：减法
  - 21-30 层：乘法
  - 31+ 层：除法
- **速度递增**: 每 5 层，方块移动速度增加 5%

### 5. 视觉反馈系统

- **方块颜色**: 根据堆叠精度改变颜色
  - 完美堆叠：绿色
  - 良好堆叠：黄色
  - 普通堆叠：橙色
  - 差堆叠：红色
- **得分特效**: 堆叠成功时显示浮动分数
- **成就解锁**: 解锁成就时显示提示

### 6. 成就系统

```javascript
const achievements = [
  { id: 'stack50', name: '堆叠大师', description: '堆叠50层', target: 50, type: 'level' },
  { id: 'math30', name: '数学天才', description: '答对30题', target: 30, type: 'math' },
  { id: 'perfect10', name: '完美主义者', description: '10次完美堆叠', target: 10, type: 'perfect' }
];
```

### 7. 本地存储

- **最高分**: 保存最高分到 localStorage
- **成就进度**: 保存成就解锁状态
- **游戏进度**: 可选保存当前游戏进度（用于恢复）

## 用户界面设计

### 主游戏区域

- **顶部**: 分数、层数、当前难度显示
- **中央**: 游戏区域，显示堆叠的方块
- **底部**: 控制按钮（开始、暂停、重新开始）

### 数学题模态框

- **题目显示**: 大字体显示数学题
- **答案输入**: 数字输入框
- **提交按钮**: 提交答案
- **反馈区域**: 显示正确/错误反馈

### 成就展示

- **成就列表**: 显示所有成就及其解锁状态
- **解锁提示**: 解锁成就时弹出提示

## 响应式设计

- **移动端优先**: 使用 vw/vh 单位和百分比布局
- **触摸事件**: 支持触摸和鼠标事件
- **屏幕适配**: 自动调整方块大小和游戏区域

## 性能优化

- **DOM 操作最小化**: 使用 transform 而不是 top/left
- **事件委托**: 使用事件委托处理点击事件
- **requestAnimationFrame**: 用于平滑动画

## 部署

游戏为纯静态文件，可直接部署到任何静态文件服务器（如 GitHub Pages）。

## 开发流程

1. 创建 `index.html` 文件
2. 实现基本 HTML 结构和 CSS 样式
3. 实现核心游戏循环（方块生成、移动、堆叠）
4. 实现数学题系统
5. 实现难度递进和视觉反馈
6. 实现成就系统和本地存储
7. 测试和优化
