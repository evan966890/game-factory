# Tech-Spec: 数学方块塔

**Created:** 2026-03-03

## Overview

### Problem Statement

如何将热门浏览器游戏 Stack Tower 的简单上瘾玩法与数学教育结合，创造一个既能娱乐又能学习的教育游戏，同时保持移动端友好和单指操作。

### Solution

开发一个纯 HTML5 游戏，核心玩法是点击放置方块堆叠建造高塔，每堆叠 5 个方块弹出一道数学题，答对获得双倍分数奖励。游戏包含难度递进（加法→减法→乘法→除法）、视觉反馈、成就系统和本地存储最高分。

### Scope

**In Scope:**
- 核心堆叠机制：点击放置方块，对齐得分，建造最高塔
- 数学题生成：每 5 层弹出一道随机数学题，难度随高度递进
- 分数系统：基础分 + 答题奖励分
- 难度曲线：加法（0-20层）→ 减法（21-40层）→ 乘法（41-60层）→ 除法（61+层）
- 视觉反馈：方块颜色渐变、堆叠动画、得分特效、答题正确/错误反馈
- 成就系统：堆叠 50 层、答对 50 题等成就
- 本地存储：保存最高分和成就进度
- 移动端响应式设计，单指操作

**Out of Scope:**
- 多人游戏或在线排行榜
- 复杂 3D 图形或精灵图集
- 音效（可后续添加）
- 用户账户系统
- 复杂的物理引擎

## Context for Development

### Codebase Patterns

- 纯 HTML/CSS/JS，无框架依赖

- 模块化 JavaScript：使用 ES6 类和模块模式
- CSS 动画用于视觉反馈
- 本地存储使用 localStorage API

### Technical Decisions

1. **单文件结构**：便于部署和分享，所有资源内联
2. **Canvas vs DOM**：使用 DOM 元素（div）堆叠，更简单，移动端性能更好
3. **数学题生成**：使用 JavaScript 随机生成题目，确保答案为整数
4. **难度递进**：基于堆叠高度切换题目类型
5. **本地存储**：使用 localStorage 保存最高分和成就

## Implementation Plan

### Tasks

1. **基础结构搭建**
   - 创建 index.html 基础模板
   - 设置游戏区域和 UI 元素（分数、高度、成就显示）

2. **核心堆叠机制**
   - 实现方块生成和放置逻辑
   - 实现对齐检测和得分计算
   - 实现堆叠动画和视觉反馈

3. **数学题系统**
   - 实现题目生成算法（加减乘除）
   - 实现答题界面和交互
   - 实现答对奖励和答错惩罚

4. **难度递进系统**
   - 根据堆叠高度切换题目类型
   - 调整方块速度和大小增加难度

5. **成就和存储系统**
   - 实现成就检测和解锁
   - 实现本地存储读写

6. **UI 和反馈**
   - 设计移动端友好的界面
   - 添加 CSS 动画和特效
   - 实现“再来一局”功能

7. **测试和优化**
   - 移动端测试
   - 性能优化
   - Bug 修复

### Acceptance Criteria

- [ ] 游戏可以在移动端浏览器流畅运行
- [ ] 单指点击操作响应灵敏
- [ ] 堆叠机制手感良好，对齐检测准确
- [ ] 数学题每 5 层弹出，难度随高度递进
- [ ] 答对获得双倍分数，答错无惩罚（仅错过奖励）

- [ ] 最高分本地存储并显示
- [ ] 视觉反馈清晰（颜色变化、动画、特效）
- [ ] 游戏有明确的“再来一局”按钮

- 视觉反馈要即时，增强成就感
- 游戏节奏由慢到快，保持心流体验
