# 技术规范：数学曲折跑

## 概述

### 问题陈述
需要创建一个教育性跑酷游戏，将数学学习与上瘾的跑酷玩法结合，让玩家在娱乐中练习数学技能。

### 解决方案
基于 Zigzag Runner 的曲折跑酷玩法，融合数学问答机制。玩家控制球在曲折路径上滚动，每前进一定距离弹出数学题，答对获得奖励，答错受到惩罚。

### 范围内
- 核心跑酷玩法：点击控制球在曲折路径上滚动
- 数学问答系统：加减乘除题目，随距离解锁难度
- 分数系统：前进距离 + 答题正确率
- 成就系统：里程碑奖励
- 本地存储：保存最高分和进度
- 移动端响应式设计

### 范围外
- 多人在线功能
- 复杂3D图形
- 音效系统（可选但非必需）
- 社交分享功能

## 技术决策

### 1. 游戏引擎选择
- **决策**：使用原生 Canvas 2D API，不使用游戏引擎
- **理由**：保持轻量级，无外部依赖，符合"一天内可完成"的约束
- **影响**：需要手动实现物理和碰撞检测

### 2. 数学题难度系统
- **决策**：基于距离解锁难度（加法→减法→乘法→除法）
- **理由**：渐进式学习，符合教育目标
- **影响**：需要动态生成题目和选项

### 3. 移动端适配
- **决策**：使用 `touchstart` 事件，禁用缩放
- **理由**：确保移动端单指操作体验
- **影响**：需要处理触摸和点击事件

### 4. 数据持久化
- **决策**：使用 `localStorage` 保存最高分
- **理由**：简单可靠，无需服务器
- **影响**：仅保存最高分，不保存游戏进度
